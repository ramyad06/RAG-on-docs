# Upwork API Support Bot

A retrieval-augmented chatbot that answers developer questions about the Upwork API,
grounded strictly in the official documentation PDF. If the answer is not in the docs,
the bot says so — it never fabricates information.

## Stack

| Layer | Choice |
| --- | --- |
| RAG framework | LangChain |
| Vector store | ChromaDB (persisted at `./chroma_db/`) |
| Embeddings | `BAAI/bge-small-en-v1.5` (local, no API cost) |
| PDF extraction | `pdfplumber` with table-row serialization |
| LLM | `llama-3.1-8b-instant` via Groq (streaming) |
| UI | Streamlit (chat layout, live reasoning trace, sidebar) |

## Setup

**Mac / Linux**
```bash
# Install uv (if you don't have it)
curl -LsSf https://astral.sh/uv/install.sh | sh

cd solution   # or: cd starter
uv sync       # creates .venv and installs all dependencies

cp ../.env.example .env
# Open .env and paste your Groq API key
# Get one free at https://console.groq.com/keys
```

**Windows (PowerShell)**
```powershell
# Install uv (if you don't have it)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

cd solution   # or: cd starter
uv sync       # creates .venv and installs all dependencies

copy .env.example .env
# Open .env and paste your Groq API key
# Get one free at https://console.groq.com/keys
```

## Run

```bash
# Step 1 — drop the PDF into solution/ or starter/ (same level as app.py)
# then build the vector index (one time, ~1 min on first run)
uv run python -m src.ingest

# Step 2 — launch the chat UI
uv run streamlit run app.py
```

Then open <http://localhost:8501> and ask a question.

## Tests

```bash
uv run pytest tests/
```

## Deploy to Streamlit Community Cloud

- App file: `app.py`
- Dependencies: `requirements.txt`
- Add this secret in Streamlit's Advanced settings:

```toml
GROQ_API_KEY = "your_groq_api_key_here"
```

The committed `chroma_db/` index means the app starts without needing the private PDF in the repo.

## Design notes

- **pdfplumber over pypdf** — the API docs contain parameter tables. pypdf flattens them into unreadable text; pdfplumber exposes structured rows that get serialized back to searchable text before embedding.
- **Chunking** — 500-char chunks with 50-char overlap. Overlap prevents OAuth steps, curl examples, and parameter tables from being cut at boundaries.
- **Two-phase retrieval** — vector similarity fetches 24 candidates; a lightweight keyword re-ranker then scores and returns the top 5. This catches exact technical terms (endpoint URLs, parameter names, TTL values) that embedding similarity can under-weight.
- **4-case prompt** — the system prompt enforces mutually exclusive response modes: greeting, direct quote, grounded inference, and a verbatim hallucination-guard fallback. The LLM never shows the case labels to the user.
- **Streaming** — answers render token-by-token via the OpenAI SDK's `stream=True`. A live reasoning trace shows retrieved chunks before generation begins.
- **`@st.cache_resource`** — the embedding model and Chroma collection load once per session, not per query.
